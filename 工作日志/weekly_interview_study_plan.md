## 第1周：定时器 / 中断 / NVIC

| 项目 | 内容 |
| --- | --- |
| 日期 | 2026-04-14 ~ 2026-04-20 |
| 本周目标 | 把你已经在轮腿项目里用过的“1ms 心跳 + 多周期执行”彻底讲透 |
| 需要搞懂 | 定时器时基、预分频和自动重装、PWM 原理、输入捕获、输出比较、中断响应流程、NVIC 优先级、抢占优先级与响应优先级、`volatile` 在中断共享变量中的作用 |

| 本周验收标准 | 你能不用资料解释为什么你的控制系统按 `1ms/5ms/10ms/20ms` 分周期执行 |

### 面试题清单

| 题号 | 题目 | 你需要讲到的点 |
| --- | --- | --- |
| 1 | 你在平衡车/轮腿里为什么要设计 1ms 基准周期？ | 控制节拍统一、避免逻辑乱序、便于多环分频执行 |
| 2 | 定时器的预分频和自动重装分别决定什么？ | 计数频率、溢出周期 |

| 4 | 输入捕获和普通定时中断有什么区别？ | 一个测外部信号时间特征，一个只是内部周期事件 |
| 5 | 中断里为什么不建议放太重的逻辑？ | 阻塞其他中断、抖动、丢实时性 |
| 6 | `volatile` 为什么经常和中断共享变量一起出现？ | 防止编译器优化导致主循环读不到真实更新 |
| 7 | NVIC 优先级怎么影响系统行为？ | 高优先级抢占低优先级，影响实时响应 |

### 面试题清单

| 题号 | 题目 | 你需要讲到的点 |
| --- | --- | --- |
| 1 | UART 和 USART 有什么区别？ | UART 只做异步；USART 可支持同步/异步，实际项目常当普通串口用 |
| 2 | UART 为什么不需要时钟线？ | 异步通信，双方约定波特率 |
| 3 | SPI 为什么速度通常比 I2C 更高？ | 推挽、全双工、时钟明确、协议开销更小 |
| 4 | I2C 为什么能挂多个设备？ | 地址机制 |
| 5 | SPI 的四种模式是怎么来的？ | CPOL + CPHA 组合 |
| 6 | 为什么 UART 中断接收比轮询更常用？ | 降低 CPU 空转，提高实时性 |
| 7 | 你在项目里是怎么判断通信故障来自主控、外设还是连线的？ | 分层排查：收发日志、IO、电源、协议响应 |

## 第4周：RS485 / Modbus RTU

| 项目 | 内容 |
| --- | --- |
| 日期 | 2026-05-05 ~ 2026-05-11 |
| 本周目标 | 把你现在对 RS485/站号/CRC 的直觉补成标准化理解 |
| 需要搞懂 | `RS485` 是物理层、`Modbus RTU` 是协议层、半双工、方向控制、主从模式、地址/功能码/数据域/CRC、异常响应、超时与帧间隔 |
| 最小实践 | 自己手写 CRC16；写一个简化版 `Modbus RTU` 收发 demo；最好能做一主一从或主机模拟 |
| 本周验收标准 | 你能明确说出“站号不是 RS485 自带概念，而是 Modbus 这类上层协议里的地址概念” |

### 面试题清单

| 题号 | 题目 | 你需要讲到的点 |
| --- | --- | --- |
| 1 | RS485 和 Modbus RTU 是什么关系？ | RS485 是物理层；Modbus RTU 是跑在其上的协议之一 |
| 2 | 为什么 RS485 适合现场多节点？ | 差分抗干扰、总线拓扑、距离相对更长 |
| 3 | Modbus RTU 一帧通常有什么？ | 地址、功能码、数据、CRC |
| 4 | CRC 在这里的作用是什么？ | 检测传输错误 |
| 5 | 半双工意味着什么？ | 同时刻不能双向都发，需控制收发方向 |
| 6 | 站号是干什么的？ | 主站区分目标从设备 |
| 7 | 如果现场经常 CRC 错或丢响应，你怎么查？ | 波特率一致性、方向切换、接线、终端、电磁干扰、时序 |

### 面试题清单

| 题号 | 题目 | 你需要讲到的点 |
| --- | --- | --- |
| 1 | CAN 为什么比普通串口更适合多节点设备网络？ | 总线仲裁、可靠性、错误处理机制 |

| 7 | 如果 CAN 通信不通，你先查什么？ | 波特率、终端电阻、引脚、过滤器、收发器、电平 |
