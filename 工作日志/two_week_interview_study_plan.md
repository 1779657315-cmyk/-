## 第1阶段：定时器 / 中断 / NVIC

| 项目 | 内容 |
| --- | --- |
| 日期 | 2026-04-14 ~ 2026-04-15 |
| 本阶段目标 | 把你已经实际用过的“1ms 基准周期 + 多周期控制”讲透 |
| 需要搞懂 | 预分频、自动重装、计数频率、定时中断、PWM、输入捕获、NVIC、中断优先级、`volatile` |
| 最小实践 | `1ms` 定时中断 demo；`PWM` 控电机 demo；输入捕获/测速 demo |
| 验收标准 | 你能不用资料解释为什么控制系统按 `1ms/5ms/10ms/20ms` 分周期执行 |

### 面试题清单

| 题号 | 题目 | 你至少要讲到的点 |
| --- | --- | --- |
| 1 | 你在轮腿项目里为什么要设计 `1ms` 基准周期？ | 统一节拍、便于多环分频、避免逻辑乱序 |
| 2 | 定时器的预分频和自动重装分别决定什么？ | 计数频率、溢出周期 |
| 3 | PWM 为什么能控电机速度？ | 改变占空比，改变平均能量/有效电压 |
| 4 | 输入捕获适合拿来做什么？ | 测频率、脉宽、编码器相关时间特征 |
| 5 | 中断里为什么不能做太重的逻辑？ | 影响实时性、阻塞其他中断、增加抖动 |
| 6 | `volatile` 为什么经常出现在中断共享变量里？ | 防止编译器优化导致主循环读不到真实值 |
| 7 | NVIC 优先级为什么重要？ | 决定响应顺序和抢占关系 |

## 第2阶段：UART / SPI / I2C

| 项目 | 内容 |
| --- | --- |
| 日期 | 2026-04-16 ~ 2026-04-17 |
| 本阶段目标 | 把你“会配会用”的串行通信补成“能系统解释” |
| 需要搞懂 | `UART` 和 `USART` 区别、波特率、异步通信、轮询与中断接收；`SPI` 主从、`CPOL/CPHA`；`I2C` 起始、停止、ACK、地址机制 |
| 最小实践 | `UART` 中断收发 demo；`SPI` 读寄存器 demo；`I2C` 读寄存器 demo |
| 验收标准 | 你能清楚说出 `UART / SPI / I2C` 的差异、优缺点和使用场景 |

### 面试题清单

| 题号 | 题目 | 你至少要讲到的点 |
| --- | --- | --- |
| 1 | `UART` 和 `USART` 有什么区别？ | UART 只做异步，USART 可支持同步/异步 |
| 2 | `UART` 为什么不需要时钟线？ | 异步通信，双方约定波特率 |
| 3 | `SPI` 为什么通常比 `I2C` 更快？ | 全双工、协议开销更小、时钟机制直接 |
| 4 | `I2C` 为什么能挂多个设备？ | 地址机制 |
| 5 | `SPI` 四种模式怎么来的？ | `CPOL + CPHA` 组合 |
| 6 | 串口中断接收和轮询接收有什么区别？ | 实时性和 CPU 占用差异 |
| 7 | 如果一条通信链路不通，你怎么先分层排查？ | 电源、连线、波形/日志、协议响应、软件逻辑 |

| 需要搞懂 | ADC 分辨率、采样时间、转换时间、连续采样、DMA 搬运、滤波、控制系统中的采样频率 |
| 最小实践 | `ADC` 单通道采样 demo；`ADC + DMA` 连续采样 demo；加一个简单滤波 |

### 面试题清单

| 题号 | 题目 | 你至少要讲到的点 |
| --- | --- | --- |
| 1 | ADC 的分辨率是什么意思？ | 量化精度，如 12 位对应 0~4095 |
| 2 | 为什么 ADC 常和 DMA 一起用？ | 连续采样、减轻 CPU 搬运压力 |
| 3 | 采样时间太短会怎样？ | 数据不稳定、采样不准 |
| 4 | DMA 本质上解决了什么问题？ | 数据搬运不占主 CPU 处理流程 |
| 5 | 控制系统里采样频率怎么定？ | 信号变化速度、控制周期、CPU 负载 |
| 6 | 模拟量为什么常要滤波？ | 抗噪声、稳定控制输入 |
| 7 | ADC 数据抖动很大时你先查什么？ | 供电、接地、采样时间、前端电路、滤波策略 |

## 第4阶段：RS485 / Modbus RTU

| 项目 | 内容 |
| --- | --- |
| 日期 | 2026-04-19 ~ 2026-04-20 |
| 本阶段目标 | 把你现在对 RS485/站号/CRC 的直觉补成标准化理解 |
| 需要搞懂 | `RS485` 是物理层，`Modbus RTU` 是协议层；半双工、方向控制、主从模型、地址、功能码、数据区、CRC、异常响应 |
| 最小实践 | 手写 `CRC16`；画出一帧 `Modbus RTU`；最好做一个简化收发 demo |
| 验收标准 | 你能明确说出“站号不是 RS485 自带概念，而是上层协议里的地址概念” |

### 面试题清单

| 题号 | 题目 | 你至少要讲到的点 |
| --- | --- | --- |
| 1 | `RS485` 和 `Modbus RTU` 是什么关系？ | 物理层和协议层关系 |
| 2 | 为什么 `RS485` 更适合工业现场？ | 差分抗干扰、多节点、距离更长 |
| 3 | 半双工意味着什么？ | 同一时刻不能双向都发 |
| 4 | `Modbus RTU` 一帧里通常有什么？ | 地址、功能码、数据、CRC |
| 5 | CRC 在这里起什么作用？ | 检测传输错误 |
| 6 | 站号是什么层面的概念？ | 主站识别目标从设备 |
| 7 | 现场 CRC 错或经常收不到响应，你怎么查？ | 接线、方向控制、波特率、干扰、帧间隔、软件时序 |

## 最后提醒

这 2 周最怕的不是学不完，而是学完了仍然停留在：

> “我知道这个东西怎么配，但我解释不清为什么。”

所以每个阶段都必须做这 3 件事：

| 动作 | 要求 |
| --- | --- |
| 1. 先问 AI 梳理框架 | 快速把概念和边界理顺 |
| 2. 再做最小 demo | 不做 demo 等于没学会 |
| 3. 最后口头回答面试题 | 不会讲清就说明还没真正掌握 |
